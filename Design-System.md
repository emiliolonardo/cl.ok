D.O.S. Time OS — Design System

Visione

D.O.S. Time OS non deve sembrare un semplice time tracker. Deve apparire come un modulo del Design Operating System di D.O.S.: preciso, nitido, tecnico, ma anche concettuale. L'interfaccia deve comunicare controllo, intelligenza operativa, modularità e cultura progettuale.

Il riferimento funzionale può essere Clockify, ma il linguaggio visivo deve essere chiaramente D.O.S.: un sistema operativo per il lavoro progettuale, non un gestionale generico.

Brand principles

1. Operating System Aesthetic

L'interfaccia deve evocare un ambiente operativo modulare, quasi infrastrutturale.
Non deve sembrare né troppo corporate né troppo giocosa.

2. Precision + Tensione Creativa

Ogni elemento UI deve risultare estremamente leggibile e funzionale, ma con una leggera tensione visuale che richiami il mondo design/innovation.

3. Sharp Geometry

Preferire angoli netti, box squadrati, griglie rigorose, allineamenti puliti. Evitare interfacce troppo morbide o arrotondate.

4. Light but Technical

Il sistema deve sembrare leggero e veloce, ma anche tecnico e affidabile.

5. Modular Layering

Ogni area deve apparire come un modulo attivabile del sistema: dashboard, timer, analytics, projects, team, reports, ABCoDE phases, mesh relations.

Regole d'uso colore

Il blu e il ciano devono essere usati con precisione, non spalmati ovunque.

Il bianco carta deve restare dominante per favorire leggibilità e aria.

Il nero profondo va usato per testo forte, aree premium, topbar o moduli in focus.

Il ciano deve funzionare come segnale di attivazione, stato live, hover o highlight tecnologico.

Il blu è il colore di struttura e identità.

Istruzioni operative per Visual Studio Code / AI coding assistant

Usa questo design system per sviluppare una web app SaaS chiamata D.O.S. Time OS, ispirata funzionalmente a Clockify ma più avanzata e coerente con l'identità D.O.S.

Obiettivo progettuale

Costruire un'interfaccia enterprise design-tech modulare, sharp, desktop-first, adatta al time tracking avanzato per team di design, innovazione e consulenza.

Regole obbligatorie

Usa una UI pulita, tecnica, squadrata, con pochissimi angoli arrotondati.

Applica la palette D.O.S.: blu #0000FF, ciano #00FFFF, bianco carta #FCFCFC, nero profondo #000016.

Usa IBM Plex Sans come font principale e IBM Plex Sans Condensed per label e tabelle dense.

Usa DOS_Pixelcity solo per logo, titoletti speciali e marker identitari.

Crea layout a sidebar + topbar + main content modulare.

Progetta i componenti come moduli di un operating system, non come semplici widget SaaS.

Mantieni forte gerarchia visiva, tanto spazio bianco, bordi sottili, ombre leggere.

Evita gradienti inutili, skeuomorfismo, glassmorphism e look troppo consumer.

Dai priorità assoluta a leggibilità, velocità d'uso e controllo operativo.

Prevedi dark mode come estensione futura, ma progetta prima una light mode impeccabile.
